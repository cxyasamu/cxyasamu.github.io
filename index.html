DOM的基本操作

DOM

Document Object Model (DOM) 中文名为文档对象模型, 它是一个编程的接口(API), 允许我们在文档中创建、修改、删除、查找元素; 我们还可以获取元素并绑定事件, 使页面更加的动态;

DOM 将 HTML 文档视为节点树, 每一个节点代表一个 html 元素; DOM 节点分为元素节点、文本节点、属性节点

查找节点

  • getElement 系列

    返回一个或多个节点集合, HTMLCollection 对象(动态 nodeList 对象)

  • querySelector 系列

    返回一个或多个节点集合, nodeList 对象(静态 nodeList 对象), 支持 css3 选择器

  • 查找父节点

  • previousElementSibling 查找前一个同级(兄弟)元素

  • previousSibling 查找前一个同级(兄弟)节点 (包含文本节点、元素节点)

  • nextElementSibling 查找后一个同级(兄弟)元素

  • nextSibling 查找后一个同级(兄弟)节点 (包含文本节点、元素节点)

  • 查找子节点

操作类名 (动态切换类名)

  • className

可读取或重置元素 class 属性

  • classList

返回元素类名列表 (DOMTokenList 对象), classList 的属性是只读的, 如果要操作这个对象, 必须使用 classList 的内置方法

属性节点

  • setAttribute(属性名，属性值) 设置元素属性

  • getAttribute(属性名) 获取元素属性

  • removeAttribute(属性名) 删除元素属性

添加节点

  • createElement(元素名称) 创建元素节点

  • appendChild(元素)

  • append(元素)

  • createTextNode(文本内容) 创建节点

  • insertBefore(插入的新节点，被指定元素之前插入)

  • replaceChild(替换的新节点，需要被替换的旧节点)

  • cloneNode(布尔值)

删除节点

  • remove() 删除自身

  • removeChild(子节点) 删除子节点

补充

  • innerHTML 设置或获取位于元素开始和结束标签内的 html

  • outerHTML 设置或获取位于元素开始和结束标签内的 html 形式(当前元素 + 元素内容)
    注意:虽然元素将在文档中被替换，设置了 outerHTML 属性的变量仍将保持对原始元素的引用

  • innerText 设置或获取位于元素开始和结束标签内的文本(元素文本内容)

  • outerText 设置或获取位于元素以及开始和结束标签内容 文本形式(当前元素+元素内文本内容，不包含元素内标签、注释节点)

事件

事件是 js 和 DOM 的交互桥梁(对事件做出反应),事件通常是和函数结合使用，通过用户操作行为(点击、手指触摸、鼠标)事件来驱动函数的执行(回调函数)

js 事件等级

  • DOM 0 级事件
  • DOM 1 级事件

1 级 DOM 标准(没有提出相关事件内容)

  • DOM 2 级事件
  • DOM 3 级事件

推出更多的事件类型,例如:blur,focus,keydown,keyup,dblclick,scroll,load,mouse,touch,drag,change…

事件流

事件流表述的页面接收事件的触发顺序，页面触发一个事件，会按照一定的顺序来响应事件，事件的响应过程就是事件流(过程)

阻止冒泡(传播)

  • event.stopPropagation()

-window.event.cancelBubble = true(IE)

阻止默认行为

例如:表单提交、a 链接跳转等，如果想要控制，必须要阻止默认行为(自带行为)

  • event.preventDefault()

  • window.event.returnValue =false(IE)

事件代理(事件委托)

事件代理的实现原理利用了冒泡阶段和目标阶段，把处理的事件添加到父元素上，等待子元素的事件冒泡，并且可以通过父元素 event.target 判断具体的子元素，从而做出处理

拖拽事件 drag

移动端事件 touch

移动事件类型

touch 事件对象属性

js堆和栈

堆和栈内存

栈内存

存放基本数据类型，占据固定的大小空间(String、number、boolean、null、undefined)

系统会自动分配内存空间，也会自动释放(GC)

堆内存

存用引用数据类型(Object、Function、Array),保存在栈内存中(变量),包含引用类型的变量实际上保存的不是变量本身，指向的是对象的引用地址;

动态分配内存空间，内存大小不一定会自动释放(循环引用)

深浅拷贝(克隆)

浅拷贝

把父对象的属性,全部拷贝子对象，如果过父对象存在引用数据类型的属性，拷贝的只是引用地址，意思就是父对象的引用地址和子对象的引用地址指向同一个内存空间

深拷贝

把父对象的属性全部拷贝到子对象,并且引用数据类型的内容相互独立(指的是内存空间也被克隆一份)

JSON 方法

JSON.parse(JSON.stringify(obj)) 深克隆缺点

  • 克隆不了函数和 undefined，在序列化的时候会丢失
  • NaN 值，在序列化的时候会变成 null
  • 只能克隆可枚举的对象属性

js对象

对象 object

javaScript 所有的事物都是可以称为对象，例如 String、Number、Array、Function 等等

现实生活中能够被称为对象的例子:

汽车 就是一个对象，汽车由零件、配色、尺寸、型号等等组成，这些汽车信息都可以称为对象的属性，汽车可以拉货、载人、比赛等等可以视为对象的方法(功能)

如何创建对象

  • 通过 new 操作符，new Object()

  • 通过 对象字面量{}

操作对象的属性

  • 通过’.’的形式，obj.属性名
  • 通过’[ ]’的形式，obj[“属性名”]
  • 通过 delete 关键字进行属性的删除,delete obj[“属性名”](delete obj.属性名)

键值对

对象遍历方法

  • for…in(推荐 object 使用，array 不推荐)
  • for…of(ES6)

对象方法

  • Object.values(对象)

    返回对象自身的所有可枚举属性值的数组

  • Object.keys(对象)

    返回对象自身的所有可枚举属性名的数组

  • Object.defineProperty

    直接再一个对象上定义新属性，或者修改对象现有的属性，并且返回此对象

  • Object.freeze(对象)

    对象被冻结，属性不可添加、修改、删除(单例模式)

  • Object.entries(对象)

    返回对象的可枚举字符串属性的键值对,返回值为数组

  • Object.assign(目标对象,…源对象)

    将一个或者多个源对象把可枚举的属性克隆到目标对象上,并返回修改之后对象，目标对象也会被修改

  • Object.create(对象)

    以现有的对象作为原型，创建一个新对象(新对象和原有对象是不同的)

js的预编译

js 的预编译

预编译分为全局预编译和函数预编译

  • 全局预编译发生在页面加载完毕的时候
  • 函数预编译发生在函数执行的前一刻(不包括没执行的函数)

回调函数

被作为一个实参传入另一个函数中，并且在该函数内部调用(执行),这种函数就称为回调函数(高阶函数);
简单理解:一个被当作参数的函数，就是回调函数

回调函数的作用

  • 函数在某个任务完成之前不运行的，在任务完成之后立即执行，它可以帮助我们完成异步编程的代码(请求数据，ajax)

  • 可以实现函数内部访问函数内部的值(执行一个函数的同时执行另一个函数)

递归函数

简单描述:函数本身调用本身

递归函数(递归算法),将问题无限拆分为多个小规模问题，然后再递归调用同一个方法解决问题

递归的注意点

每一个递归函数都需要一个终止条件(自己查找递归的规则),否则就是”死递归”(栈溢出、爆栈)

递归的三步走

1.写出递归公式(“找到每次函数执行的结果与上一次执行结果的关系，相当于是反推计算”)

2.确定终止条件(可以为多个)

3.return 输出结果(和上一个结果进行操作)

js函数

函数

函数是完成某个功能的一组语句，定义函数之后，函数体里面的代码可以重复使用(定义函数)

函数的参数

在调用函数的时候，需要传递参数到函数内部，就要使用函数的参数传递的操作，函数内部存在两个隐藏参数(默认)，分别是 this 和 arguments

  • this (函数的执行上下文对象)，以函数形式调用，this 指向的是 window 对象

  • arguments 是一个类数组(无数组的方法、有数组的长度)，可以通过索引/下标值操作数据以及获取长度，length 属性当前实参的个数，callee 属性对应的是函数本身

js 执行环境(执行上下文)

决定上下文定义了变量或者函数有权访问其他的数据，决定了他们各自的行为(js 出现多个执行环境)

  • 什么是作用域?

它指的是变量或函数具有访问权限的代码区域;在 js 中，作用域是在函数中维护的，表示变量或者函数起作用的区域，例如函数中的作用域指代的是在函数执行上下文中、全局作用域指代的是全局的执行上下文

  • 全局作用域

声明在函数外部的变量、函数，在代码中的任何地方都可以访问到

  • 函数作用域(局部作用域)

声明在函数内部的变量、函数，局部作用于固定的代码片段中可以访问(代码片段就是函数执行代码区域)

  • 块级作用域(ES6 提出)

使用 let/const，在{}里面的代码就会产生块级作用域，例如 for、if、while

作用域链

函数内部如果在当前的作用域中查找不到对应的变量/值，自动会向上一层作用域进行查找，直到查找到全局的作用域，如果还找不到就会直接报错 xxx is not defined;这个过程会形成一条查找链条，这个链条就是作用域链
