BFC

BFC 概念

块级格式化上下文(block formating context)，属于普通流(web 页面 css 渲染的部分),可以看作是一个页面的区域，它拥有属于自己的渲染规则，不会影响到其他元素的布局。

BFC 特征

  • 每一个 BFC 区域只包含子元素，不包含其他子元素的子元素

  • 每一个 BFC 区域是相互独立，互不干扰

BFC 触发条件(只需满足一个条件即可)

定位

定位

属于 html 和 css 的布局方式,其他的布局方式:流式布局(百分比布局)、弹性布局(flex)、浮动布局(float)、定位布局在涉及到多个元素层叠的情况下，推荐使用定位布局

绝对定位 absolute

相对于当前元素(不为静态定位)的父元素定位,如果当前元素不存在定位,往上一层元素查找，直到查找到不为静态定位元素位置，最终还找不到，那么默认从屏幕的最左上角进行定位(根元素定位)，可以通过设置 left、top、bottom、right 进行位置调整

注意:不可以和浮动一起使用，float 会失效

相对定位 relative(正常文档流)

相对自身元素所处位置进行定位，可以通过设置 left、top、bottom、right 进行位置调整(类似 margin 属性)，移动后之前的位置会一直保留(占据)

固定定位 fixed(脱离文档流)

相对于浏览器窗口进行定位，可以通过设置 left、top、bottom、right 进行位置调整(默认层级最高)

粘性定位 sticky(未到达粘性值不会脱离文档流)

设置粘性定位的元素默认情况下为相对定位,当滚动到粘性值的时候,会切换为固定的固定定位,直到父元素脱离屏幕(消失)

层叠级别 z-index(定位不为静态定位时使用)

设置定位元素层叠级别，默认 0，值越高层级越高(如果层级相同的情况下,默认后面的元素会覆盖到最上层)

浮动

浮动

设置一个元素发生漂浮，浮动后元素默认大小为内容的大小，并且可以设置宽高，也可以和其他浮动元素或者行元素共处一行

什么是浮动布局

浮动的框(元素)可以向左右方向移动，直到他的外边缘包含框或者另一个浮动边框为止

浮动属性

浮动的影响

浮动元素会脱离文档流，导致父盒子的高度塌陷(高度丢失，前提是没有设置高度)，不仅仅对当前父盒子造成影响，还会影响到父盒子以外的布局情况

文档流

元素一般存在两种情况，一种是普通流(文档流)，另一种就是脱离普通流

普通流，就是文档显示对象在排列时所占的位置，可以将网页看作是一个文档，这个文档分层次，每一行中的内容从左到右依次排列

脱离普通流，设置浮动、绝定定位等操作就使当前元素脱离普通流，此时元素不存在普通流中，而是漂浮在普通流的上方，能够左右移动并且不会影响到普通流

如何解决父盒子高度塌陷问题

  • overflow 解决(BFC)
  • display 解决(BFC)
  • clear : both; 解决(after 伪元素、空标签)

清除浮动 clear

用于清除元素浮动(用于指定元素的左/右侧不允许存在其他浮动元素)

盒子模型

盒子模型

什么是盒子模型

所有的 html 元素都可以看作是一个盒子，这个盒子包括：内外边距、边框、实际内容，一个完整的盒子模型由 width、height 组成的内容区、border 边框区、padding 内边距区、margin 外边距区

W3C 盒子模型(标准盒模型)

由 width、height 组成的内容区、border 边框区、padding 内边距区、margin 外边距区组成

IE 盒子模型(怪异盒模型)

width 由左右 border、内容宽度、左右 padding，height 由上下 border、上下 padding、内容高度组成

内外边距(padding、margin)

外边距用于定义元素的周围空间(间隙)，内外边距是控制当前元素和其他内部元素之间的距离

边框 border

设置边框颜色、样式、宽度

盒子阴影 box-shadow

文字阴影 text-shadow

选择器

选择器

元素选择器

通过指定元素名称去设置样式

群组选择器

可以将多个选择器通过’,’连接多个选择器的样式,减少代码的冗余

通配选择器

能够匹配所有元素的选择器(重置浏览器默认样式或者 html 标签自带样式)

类名选择器

别名 class 选择器,通过元素的 class 全局属性去设置样式

ID 选择器

通过元素的 id 全局属性去设置样式,注意一个 ID 选择器只能存在一个元素的 id 属性上(规范),最好不要在 id 选择器上设置样式

选择器之间的权重(优先级)

  • !important 无视优先级
  • 行内样式 优先级 1-0-0-0
  • ID 选择器 优先级 0-1-0-0
  • 类选择器 优先级 0-0-1-0
  • 元素选择器 优先级 0-0-0-1
  • 通配选择器 优先级 0-0-0-0

后代选择器

语法需要使用空格去匹配当前元素内部的选择器

子代选择器

语法需要使用 > 去匹配当前选择器的子代选择器

兄弟选择器

语法需要使用 ~ 去匹配当前同级别的选择器(向下同级)

相邻选择器

语法需要使用 + 去匹配当前选择器相邻的选择器(向下相邻)

伪类选择器

  • 结构类型
  • 用户行为类型

伪元素选择器

在当前元素插入虚拟的元素结构，例如装饰前置和后置。首字母大写操作

  • 装饰首字母 ::first-letter
  • 修饰元素的前置/后置 插入虚拟元素(不存在 html 结构中，只存在 css 中，只能给非置换元素设置)

块级元素和行内元素

  • 块级元素

    • 独占一行，宽度由客户端屏幕大小决定
    • 可设置宽高、内外边距

  • 行内元素

    • 可以和其他行内元素共占一行，宽度由元素内容控制
    • 不可设置宽高，左右外边距有效、内边距有效(置换元素除外)
  • 置换元素

    内容不受 css 视觉控制(css 渲染模型)，默认本身就存在固定的尺寸(宽、高、宽高比)

css 基础

css 基础

中文名:层叠样式表,是属于计算机语言(表现 html 文件),主要作用是装饰 html 标签,还可以负责页面的布局(主要)

css 历史

  • 1994 年，第一个 css 提案的发布(web开始流行)，css 的诞生
  • 2001 年，微软公司发布 IE6 浏览器，该浏览器拥有独特的渲染机制(困扰css的兼容性)
  • 2014 年，微软公司发布 IE10 浏览器，兼容性有所优化(针对兼容性问题，发布了部分的预处理器 — sass、less；后处理器 — postcss)

css 的特点

  • 表现与结构分离

    将 css 样式写道对应的 css 文件中，html 文件单独负责页面结构(结构清晰简洁)

  • 支持多设备样式

    pc、移动、ipad、超大屏

  • 易于维护和改版

    只需要操作 css 文件，那么对应的 html 文件的内容会自动更新

  • 简化设计开发

    css 的样式可以通过继承的方式去使用(层叠效果),不需要写重复的样式

    css 语法的结构

    选择器和声明快，声明块包括对应 css 属性和值，声明块可以拥有多个属性

引入 css 样式

  • 行内/内联样式(不推荐)(权重最高,其他的样式会被直接覆盖,遵循就近原则)

  • 内部/页内样式/嵌入式样式

  • 外部/外链样式(推荐)

css 三大特征

  • 继承性

    子元素会继承父元素的某些样式，例如:文本颜色、大小、文本对齐方式、字体族

  • 层叠性

    相同的选择器给同一个元素设置不同/相同样式的时候,如果是相同样式，后者会覆盖前者的样式，如果存在不同样式，直接层叠/叠加样式

  • 优先级(权重)

    选择器相同，执行层叠性;选择器不同，根据选择器的权重执行
