js的类型判断和转换

数据类型的判断

typeof

用于判断数据类型(比较适合用于判断基础数据类型),返回当前数据的类型(字符串)；typeof 检测规则是把当前数值类型进行二进制转换,前三位二进制数值为 0，统一标识为对象(js 数据类型在底层)

为什么 null 会错误的识别为 object 的主要原因(一个历史遗留问题)

instanceof

用于检测构造函数的原型对象属性是否出现在某个实例对象的原型上，返回的布尔值

Object.prototype.toString.call()

使用对象的原型去判断数据类型(经常使用的方法)

数据类型的转换(强制类型转换)

  • String(数据)全局方法(任意 js 环境下调用)

    将其他数据类型转换字符串类型，返回的是字符串类型的数值，String 函数中的参数会调用 toString 方法进行字符串转换

  • toString()对象方法(只有在当前对象下才能使用)

    和 String 方法的区别:不支持 null 和 underfined 的类型转换(报错，查找不到方法)

  • Number()全局方法

    将其他数据类型转换数字类型

  • Boolean()全局方法

    将其他数据类型转换为布尔类型,返回 true/false

数据类型

各语言的原始数据类型(字符串、数字、布尔值),js 属于弱类型语言

字符串 String

可以使用双引号或者单引号表示字符，string 类型表示 0 或者多个 Unicode 字符组成的字符序列，统称为字符串

  • 转义字符

    字符串存在特殊符号，用于表达非打印符号，这种就是属于转义字符

  • 字符串长度

    档期啊字符的长度，通过字符串自带的 length 属性获取

  • 字符串拼接

    使用’+’进行字符串拼接

支持整数、负数、小数，js 的数字类型使用的是双精度浮点型(64 位二进制)，表达方式两种，整数和浮点数(特殊值:NaN(not a number)、Infinity、-Infinity)

布尔值 Boolean

字面值:true || false 其他表示形式:0、NaN、空字符串、underfined、null 他们的布尔值都是 false

未定义值 Underfined

声明变量但是没有对齐赋值，表示无值(空的值);
函数默认返回值也是 underfined;
访问对象不存在属性的时候，返回 underfined

空对象 Null

空值，空的对象，空/不存在的对象引用地址(指针)

唯一值 Symbol

表示独一无二的值，使用方式实在定义对象唯一属性名(ES6 新增数据类型)

引用数据类型(复杂数据类型)

对象类型 Object

对象是数据和功能的集合，对象的数据存放方式以键值对形式(键名和键值)

js入门

js 入门

概述

js 全称 jacascript，是一门浏览器脚本语言，广泛应用在 web 页面开发，主要功能使当前网页添加动态向效果(UI 交互),目的为了让用户有更好的浏览体验

单线程语言

js 就是一门单线程语言，作为浏览器的脚本语言，js 主要的用途就是用来做用户交互的，操作 html 结构，决定了它就是一门单线程(每次只做一件事 不能同步进行)

多线程语言，处理多个任务(同时做多个任务)，类似 Java、c++、python

js 三大主体

  • ECMAScript(ECMA 组织定制一套标准)
  • DOM (document object model 文档对象模型)
  • BOM(browser object model 浏览器对象模型)

js 引入方式

  • 内嵌式
  • 行内式
  • 外链式

移动端

移动端知识

移动端

物理像素

每一块屏幕都是由一个一个的方块组成，这些方块名字就是物理像素点(设备像素、屏幕分辨率)，以 iPhone6 作为例子，屏幕的分辨率 750*1334，指屏幕的宽度(水平)有 750 个像素点，高度(垂直)有 1334 个像素点

css 像素

css 中像素叫做逻辑像素，也可以叫做设备的独立像素

viewport 视口

浏览器显示页面大小内容的屏幕区域就叫做视口，它们分为三种视口

布局视口 layout viewport

布局视口是页面布局的基准窗口，pc 端中布局视口等于当前浏览器的窗口大小; 移动端中布局视口被赋予一个默认值，这个值指的是设备的逻辑分辨率(可能会大很多)，所以 pc 端在手机上显示，内部元素会看起来很小，不利于观看;

针对移动端浏览器都会设置一个默认的 viewport 元标签(虚拟的布局视口)，默认大小为 980px(布局视口 iPhone6 作为例子)

为什么是 980px? 以前 pc 的网页版心是以 980 为主，设置 980 布局视口为了在移动端经量能够看到当前网页的主要内容

视觉视口 visual viewport

就是手机屏幕所能看到的区域(不需要滑动屏幕)，当用户对浏览器进行缩放，不会改变布局视口大小，改变的只是视觉视口

理想视口(完美视口) ideal viewport

为了使网页在移动端有最理想的浏览和阅读宽度而设置的，使布局视口和视觉视口宽度保持一致(比例相同)，需要手动设置 viewport 元标签(meta 标签)

布局视口的宽度 等于 设备的逻辑像素的时候，这个就是理想视口(完美视口)

注意理想视口的宽度 = 设置横向分辨率 / DPR (设备像素比)

设置视口 (设置 meta)

媒体查询 (响应式布局)

根据不同设备、不同尺寸对页面元素设置不同的样式，用于一个网站适配 pc 端、ipad 端、移动端，页面会根据媒体查询设定的宽度重新布置样式

动画

animation 动画

css 动画和过渡动画相同点:可以控制元素从初始阶段到最终阶段内发生的变化，例如动画时长、动画效果、动画的等待时间

css 动画和过渡动画的不同点:动画对元素变化的方式有更多的控制和选择，例如控制动画开始到结束之间的效果，过渡动画是控制不了的，一旦开始就无法停止，动画还可以使用 js 控制动画暂停和开始

关键帧

帧，就是动画中最小的影像画面，相当于是电影胶片的每一个镜头

animation 属性(复合属性)

想要元素拥有动画效果，必须填写此属性，并且还需要加上关键帧动画名

animation-name 动画名称

animation-duration 动画时长

animation-direction 动画播放方向

animation-iteration-count 动画播放次数

animation-delay 动画延时播放

animation-timing-function 动画频率(速度曲线)

animation-play-state 动画状态

animation-fill-mode 动画填充模式

逐帧动画 steps()

又名定格动画，它的原理就是根据不同的图像连续播放，从而产生动画效果(每一个图像代表的就是一帧)

transform 形变

主要用于元素平移、旋转、缩放, 对于变形的元素, 它会有自己的堆叠上下文, 经过缩放、平移等形变外型看似变化了, 但是实际占用的大小位置是不变的

translate 单独设置水平或者垂直方向

rotate 旋转

rotate 单独设置水平或者垂直方向

元素中心点 (变形的原点)

scale 缩放

scale 单独设置水平或者垂直方向

skew 扭曲

skew 单独设置水平或者垂直方向

透视 perspective

开启 3d 视觉效果, 配合 transform 的属性使用, perspective 类似电影的投屏, 视距越大, 当前屏幕距离 3d 物体越远

过渡效果 transition

是一个元素的过渡控制在一段时间内由初始状态转变成结束状态, 可以控制元素的过渡时间、速度曲线

flex

flex 弹性盒子(flex 布局)

css3 的新布局方式，是一种当前页面需要适应不同屏幕大小，以及设备类型时,确保元素拥有合适的行为进行布局方式

flex 特点

  • 弹性盒子由弹性容器和弹性子元素组成
  • 弹性容器设置 css 属性 display:flex 即可定义为弹性容器
  • 弹性容器中可以包含一个或者多个弹性子元素
  • 如果子元素的高度没有设置，高度则由弹性盒子是默认分配的

定义 flex 容器的主轴 flex-direction

该属性设置弹性容器的主轴方向

弹性子元素换行 flex-wrap

宽度不够所有子元素排列的时候，可以设置换行排列

弹性容器主轴方式子元素的布局方式 justify-content

交叉轴的子元素排列方式(单行子元素) align-items

交叉轴的子元素排列方式(多行子元素) align-content

定义指定的 flex 子元素交叉轴对齐方式 align-self

缩减因子 flex-shrink

如果当前的 flex 盒子宽度不够的子元素排列的情况下，会缩减子元素的宽度，缩减因子设置的值按比例缩减宽度

增长因子 flex-grow

弹性容器出现多余空间，设置 flex 子元素的增长因子，默认值为 0，不能为负数，值越大，增长值就越高

定义 flex 子元素的宽度 flex-basis

定义当前 flex 子元素是缩放因子/增长因子之前的宽度，flex-basis 优先级比 width 高(覆盖)

复合语法 flex

排序 order

用于改变 flex 子元素的排序位置
